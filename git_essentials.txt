# Basic workflow
git clone <repo-url>
git add .
git commit -m "message"
git push origin main

# Branching
git branch feature-x
git checkout feature-x
# or combined:
git checkout -b feature-x

# Merging
git checkout main
git merge feature-x

# Stashing (save uncommitted work)
git stash
git stash pop

# Viewing history
git log --oneline --graph

# Undoing things
git reset --soft HEAD~1  # undo commit but keep changes
git revert <commit-hash>  # safe undo for pushed commits

Best Practices for DevOps
Commit Messages: Clear, conventional commits

.gitignore: Exclude secrets, binaries, temp files

Small, Atomic Commits: One change per commit

Branch Protection: Require PR reviews, status checks

Semantic Versioning: Tag releases properly

Secret Management: NEVER commit secrets to Git

Infrastructure Changes via PR: All changes reviewed

Automated Testing: Pre-merge validation
